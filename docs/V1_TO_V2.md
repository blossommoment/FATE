# FATE v1.0 → v2.0 重大改动

> v1.0：Codex 生成的概念原型（桌面 `fate_v1.0` 文件夹）——FastAPI + 硬编码演示数据 + 一张静态页，用于展示产品愿景。
> v2.0：可上线的完整产品——Next.js 16 全栈，确定性八字引擎 + 双报告体系 + 传播裂变基建。

## 一、底层引擎（从"假数据"到"真排盘"）

| | v1.0 | v2.0 |
|---|---|---|
| 排盘 | 无（写死的演示命盘） | lunar-javascript 专业排盘：四柱、藏干、纳音、十二运、节气，晚子时归日（setSect 2） |
| 大运 | 无 | tyme4ts 童限起运，顺逆排、八步大运、流年索引 |
| 十神 | 无 | 藏干本气/中气/余气分层加权，月令权重最高 |
| 主格判定 | 无 | 月令主导定格 + **三合/三会成局加权定格**（三合+7/三会+9），命中时标注定格依据；副轴支持**双副轴并立**（同组十神互斥） |
| 人格建模 | 标签库静态文本 | 四维人格 + 十二维深度分析（四类）+ 专项观察（华盖/十灵日/桃花）+ 社交行为模式（多信号综合定档） |
| 确定性 | — | 同八字任何设备同结果；vitest 黄金用例锁定排盘与匹配输出 |

## 二、双人合盘（v1 完全没有）

- 六维行为模型打分（引力/情绪承接/表达译码/主导权/日常黏合/冲突修复），恋爱/朋友/同事/家人四套权重
- 跨盘结构分析：六冲、六合、三合、三会、天干相制（五行意象表达），双盘连线图可视化
- **关系判词**：相爱相杀/高山流水/榫卯相合等十一种命格组合，附幽默解读与判据
- 相处指南：行为断语（谁主动/在意浓度/氛围/服软次序/依附程度）、双方性情释读、摩擦情境与解法、长线经营——全部附命盘依据，确定性变体池避免千人一面
- 缘分签：流月干支（按节气）+ 双人五行补弱的三签分工（当令/补益/机缘），每月自动更换

## 三、报告结构（从"一张页面"到"两本书"）

- 关系剧本：目录 + 七章（关系总览/两人底色/八字化学反应/相处样态/摩擦与化解/长线经营/总结与要点），每章独立链接可转发
- 深度分析：目录 + 六章（四类维度 + 专项观察 + 社交模式）
- 特殊流年点：桃花年、天克地冲、本命之年、三合成局年，流年条角标 + 分析解读

## 四、传播与留存基建（v1 无）

- 邀请合盘链接：生辰编码进 URL，TA 填自己生日即见同一份合盘，零注册零数据库
- 分享卡片：canvas 本机生成，判词 + 幽默解读作钩子，柒章目录留悬念
- 微信/小红书 OG 链接预览（「XX」想和你合一盘）
- 本机历史记录（localStorage，按人去重）
- 排盘小面板：结果页内展开收起，不跳页

## 五、AI 层（从无到有，且有约束）

- DeepSeek/SiliconFlow 关系助手：只解释规则引擎结果，不重算命，无 key 时回退本地规则
- 标准化契约（docs/REPORT_STANDARD.md）：buildRelationshipFacts 事实清单 + AI 三条铁律，实现"事实一致、措辞可变"
